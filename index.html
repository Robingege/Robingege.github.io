ajax 无刷新技术

发表于 2017-10-19 | 分类于 学习

ajax无刷新技术的理解：

ajax技术是属于浏览器的技术，由浏览器实例化一个ajax对象，然后帮我们发送一个http请求，并携带一定的参数，当服务器端接收该请求后，对参数进行过滤，做出相应的逻辑判断，然后决定是否需要数据库的参与，做出相应的处理后，对返回的数据进行格式化（一般都处理为json格式的数据）后返回给前端，前端在接收到后端的响应后，然后做出相应的逻辑判断，通过DOM选取页面的局部区域进行数据的操作，实现局部的更新，达到页面整体没有刷新（也就是URL地址不变的情况下），局部的区域实现更新。一般用作ajax无刷新分页展示数据，用户名、邮箱的校验等功能。

好处：1. 用户的体验比较好；2. 节省服务器的宽带（减少服务器的压力）。

弊端：1. 浏览器无法记住访问的历史信息；2. 搜索引擎不利于SEO.

（二）shell & git

发表于 2017-10-09 | 分类于 学习

shell

  • shell就是我们与计算机交互所使用的东西
  • shell分为两种
    • 图形界面shell
    • 命令行shell
  • 所有的操作系统否会内置命令行shell，图形界面那就不一定了，所以有必要了解一些
    • 不同的操作系统内置的shell命令多少会有一些不一样
    • window默认内置的咱们可以称为叫dos-shell
    • 咱们学习的是bash-shell

shell命令

cd

  • 作用：切换目录
  • 切换盘符：cd 盘符:
  • 切换目录：cd 路径

ls

  • 作用：查看目录文件信息
  • 使用：ls 路径
  • 包含隐藏文件：ls -a
  • 列表的方式查看：ls -l

pwd

  • 作用：查看当前所在路径

mkdir

  • 作用：创建文件夹

touch

  • 作用：创建文件

cp

  • 作用：复制文件
  • 使用：cp 文件名 路径
  • 复制文件夹：cp -r 目录名 路径

mv

  • 作用：移动文件
  • 使用：mv 文件名 路径
  • 移动文件夹：mv -r 目录名 路径

rm

  • 作用：删除文件
  • 使用：rm 文件名 路径
  • 删除文件夹：rm -r 目录名 路径

cat

  • 作用：查看文件内容

less

  • 作用：查看文件内容

clear

  • 作用：清屏

echo

  • 作用：打印

> || >>

  • 作用：把输出文本重定向到指定文件中

  • ：会覆盖文件内容

  • ：追加文件内容

vi编辑器

三种模式

  • 命令模式
    • 在这种模式下，可以输入一些特殊命令快速的操作文本内容
  • 编辑模式
    • 向普通的记事本那样编辑书写内容
  • 底行模式
    • 也是运行一些命令，但是这些命令我们可以看到，同时需要回车执行

三种模式的切换

  • 只能从命令模式切换到编辑模式或者底行模式
  • 命令 => 编辑：i I a A o O
  • 命令 => 底行：shift + :
  • 编辑、底行 => 命令：esc

命令模式的常用命令

  • 复制：yy
  • 粘贴：p
  • 撤回：u
  • 删除一行：dd
  • 删除一个字符：x
  • 保存：ZZ

底行模式的常用命令

  • 保存：w
  • 退出：q
  • 保存退出：wq
  • 撤销修改强制退出：q!
  • 设置行号：set nu

git

git的三个工作区

  • 工作区
    • 是我们开发人员维护的
  • 暂存区
    • git自动维护的，只能自己使用与查看
  • 版本区
    • git自动维护的，将来可以共享给他人

git的四种文件状态

  • 未追踪状态
  • 修改状态
  • 暂存状态
  • 版本状态

一次性全局配置

  • git config –global user.name 你的大名
  • git config –global user.email 你的邮箱

git其他命令

  • git status
    • 查看当前的文件状态
  • git log
    • 查看提交的历史版本记录

分支操作

  • 创建新分支：git branch 新分支名
  • 切换分支：git checkout 分支名
  • 合并指定分支到当前分支：git merge 被合并的分支名
  • 删除分支：git branch -d 分支名
  • 查看所有分支：git branch

.gitignore

  • 作用：用来配置要git忽略管理的文件
  • 使用：要忽略什么文件，就把什么文件的名字写上去即可，里面也可以使用*号通配符，还可以使用！取反

仓库创建

  • git init
    • 初始化一个新的git仓库，初始化后会在你的目录下创建一个.git隐藏文件夹
    • 一个项目只需初始化一次，同时一个项目如果已经有了.git文件夹就不要再初始化了
  • git clone
    • 依据一个远程仓库创建一个本地仓库，这个仓库本身就已经初始化好了，后续就不需要再clone或者ini了。
    • 一个本地仓库只需clone一次。

远程仓库

  • git pull origin master
    • 从远程仓库拉取
  • git push origin master
    • 上传本地版本到远程仓库
  • git remote -v
    • 查看所有的远程仓库别名

版本提交

  • git add
    • 把当前文件的版本备份到暂存区
  • git commit
    • 把暂存区的文件版本移到版本区
  • git commit –amend
    • 覆盖掉最后一次的版本提交，就是给了你一个重新提交的机会

文件对比

  • git diff ：工作区与暂存区对比
  • git diff 版本ID ：工作区与版本区对比
  • git diff –cached ：暂存区与版本区对比

撤销

  • git checkout – file
    • 取消工作区的修改，本意是从暂存区或者版本区提取某文件覆盖掉工作区
  • git reset HEAD
    • 取消暂存区的修改，本意是把暂存区的内容回退到最新版本
  • git reset HEAD –hard
    • 取消暂存区与工作区的修改，本意是把暂存区与工作区的文件，回退到最新版本

一定要记住的命令

  • add
  • commit
  • pull
  • push
  • status
  • clone
  • diff
  • checkout
  • branch
